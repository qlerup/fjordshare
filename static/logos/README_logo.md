# fjord3D Logo Package

This package contains the full **fjord3D** logo set for web, favicon, PWA, and iPhone/iOS home screen use.

The files are grouped into four folders:

- `logos/` – main brand logos for your app, website, header, splash screens, and marketing use
- `icons/` – icon files in different sizes for favicons, app icons, and platform-specific use
- `web/` – ready-to-use files you can copy directly into your app's `public/` or `static/` folder
- `source/` – source/master PNG versions that the rest of the pack is based on

---

## Recommended quick start

If you only want the most important files, use these:

- `logos/fjord3D-logo-horizontal-transparent.png` → best for your app header/navbar
- `logos/fjord3D-logo-horizontal-light.png` → best on white/light backgrounds
- `logos/fjord3D-logo-horizontal-dark.png` → best on dark backgrounds
- `icons/app-icon-1024x1024.png` → master icon for apps and exports
- `web/favicon.ico` → browser tab icon
- `web/apple-touch-icon.png` → iPhone / iPad “Add to Home Screen” icon
- `web/site.webmanifest` → PWA/web app manifest file

## Folder guide

## 1) `logos/`
Main logo files.

### `logos/fjord3D-logo-horizontal-light.png`
**What it is:** The horizontal logo designed for light backgrounds.
**Best use:** Website header, web app top bar, Docker app login page, documentation, and general branding on white or light grey backgrounds.

### `logos/fjord3D-logo-horizontal-dark.png`
**What it is:** The horizontal logo designed for dark backgrounds.
**Best use:** Dark mode UI, dark splash screens, dark login screens, and hero sections with a dark background.

### `logos/fjord3D-logo-horizontal-transparent.png`
**What it is:** The horizontal logo on a transparent background.
**Best use:** The most flexible version. Great for overlays, websites, headers, presentations, custom backgrounds, and image editing where you do not want a white box around the logo.

### `logos/fjord3D-logo-mark-transparent.png`
**What it is:** The icon/mark only, without the word “fjord3D”, on a transparent background.
**Best use:** Compact branding, avatar/profile image, menu icon area, app shortcuts, favicons, badges, or places where the full logo is too wide.

### `logos/fjord3D-logo-mark-white-bg.png`
**What it is:** The icon/mark only on a white background.
**Best use:** Quick use in documents, reports, and places where a flat white background is preferred.

### `logos/fjord3D-logo-stacked-light.png`
**What it is:** Stacked logo with icon above the text, optimized for light backgrounds.
**Best use:** Landing pages, centered branding, posters, startup/loading screens, profile headers, and places where vertical layout works better than horizontal layout.

### `logos/fjord3D-logo-stacked-transparent.png`
**What it is:** Stacked logo on a transparent background.
**Best use:** Flexible centered branding on custom backgrounds and layouts.

### `logos/fjord3D-app-icon-master.png`
**What it is:** High-resolution square master app icon.
**Best use:** Main source file for app icon exports, platform resizing, app store mockups, and future conversions.

## HTML tags for web use

Copy the files from `web/` into your app's `public/` or `static/` folder, then add this in your HTML `<head>`:

```html
<link rel="icon" href="/favicon.ico" sizes="any">
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
<link rel="manifest" href="/site.webmanifest">
<meta name="theme-color" content="#002B4C">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-title" content="fjord3D">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
```

## Notes

- The **horizontal logo** is best when you have enough width.
- The **stacked logo** is best when you need a centered or vertical layout.
- The **mark/icon only** version is best for small spaces.
- The `web/` folder is the easiest one to deploy directly.
- The `source/` folder is mainly for backup and future editing/reference.
